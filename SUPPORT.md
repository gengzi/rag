# Support

Thanks for using or evaluating `gengzi/rag`.

This project is an early-stage Spring AI based RAG and Agent platform for the Java ecosystem. The best way to get help is through GitHub Issues.

## Where to Ask Questions

Use GitHub Issues for:

- Setup questions
- Bug reports
- Feature requests
- Documentation gaps
- Module design questions
- RAG / Agent / GraphRAG / MCP integration questions

## Before Opening an Issue

Please check:

- `README.md`
- `README.zh-CN.md`
- `README.en.md`
- `docs/ARCHITECTURE_OVERVIEW.md`
- `ROADMAP.md`
- Existing GitHub Issues

## Useful Information to Include

When asking for help, include:

- Target module, such as `rag-core`, `rag-chat`, `rag-agent`, `rag-graph`, `rag-mcp`, or `rag-agent-teams`
- What you are trying to build
- Expected behavior
- Actual behavior
- Local environment information
- Relevant logs or screenshots

## Project Direction

The project focuses on practical engineering patterns for Java / Spring AI teams:

- Enterprise RAG applications
- Streaming chat with SSE
- Message persistence and replay
- Agent workflow execution
- GraphRAG with Neo4j
- MCP memory and tool integration
- Multi-agent collaboration examples

Contributions that improve documentation, local startup, tests, examples, security hardening, and production readiness are welcome.
