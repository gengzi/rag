# Promotion Guide

This document provides copy-and-share materials for introducing `gengzi/rag` to developers, communities, and potential contributors.

## One-Line Description

### English

A Spring AI based multi-module RAG and Agent platform for Java backend teams.

### 中文

一个面向 Java / Spring AI 后端团队的多模块 RAG 与 Agent 工程化平台。

## Short Description

### English

`gengzi/rag` is an Apache-2.0 open-source, Spring AI based RAG and Agent platform. It covers knowledge ingestion, vector retrieval, SSE streaming chat, message persistence, Agent orchestration, GraphRAG with Neo4j, MCP memory/tool integration, and multi-agent collaboration examples.

### 中文

`gengzi/rag` 是一个 Apache-2.0 开源的 Spring AI 多模块 RAG 与 Agent 平台，覆盖知识入库、向量检索、SSE 流式对话、消息持久化、Agent 编排、Neo4j GraphRAG、MCP 记忆/工具扩展和多 Agent 协作样例。

## Why It Matters

### English

Most RAG and Agent projects are Python-first or demo-oriented. `gengzi/rag` focuses on Java / Spring AI backend engineering and provides a practical reference for enterprise teams building production-oriented AI applications.

### 中文

很多 RAG / Agent 项目偏 Python 或 Demo，本项目更关注 Java / Spring AI 后端工程落地，适合企业团队参考如何组织 RAG、Agent、GraphRAG、MCP、SSE、记忆和多 Agent 协作能力。

## Ethical Growth Notes

Do:

- Ask real developers for feedback.
- Share the project in relevant communities.
- Write useful articles and tutorials.
- Create runnable demos and screenshots.
- Welcome issues and pull requests.

Do not:

- Buy stars.
- Use fake accounts.
- Misrepresent adoption metrics.
- Claim production maturity before the project is ready.
